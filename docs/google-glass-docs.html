Glass Explorer Edition - Platform Overview

Build Glassware that is available at a user's glance on Glass. Glass is fundamentally different than existing mobile platforms in both design and use.

On This Page

Introduction to Glass

Google Glass is a wearable computer with an optical head-mounted display that displays information in a smartphone-like hands-free format. Glass communicates with the Internet via natural language voice commands.

Platform Architecture

Glass runs on Android and provides two main development paths:

Glass Development Kit (GDK)

Build Glassware that runs directly on Glass. The GDK is an add-on to the Android SDK that lets you build powerful interactions with low-level hardware access.

  • Direct hardware access
  • Real-time interactions
  • Offline functionality
  • Complex UI controls

Mirror API

Build web-based services that interact with Glass. The Mirror API provides a cloud-based API that doesn't require Android development.

  • Platform independent
  • Automatic syncing
  • Simple timeline integration
  • RESTful interface

Development Options

When developing Glassware, choose the approach that best fits your use case:

Use the GDK when you need:

  • Real-time user interaction
  • Access to hardware like the camera or sensors
  • Offline functionality
  • Complex animations or graphics

Use the Mirror API when you need:

  • Send notifications to users
  • Simple timeline cards
  • Basic media and menu items
  • Integration with existing web services

The Timeline

The timeline is the main user interface that is exposed to users and is comprised of 640 × 360 pixel cards. It provides a standard way to present information and actions to users.

// Example: Creating a timeline card with GDK Card card = new Card(context); card.setText("Hello Glass!"); card.setFootnote("My First Glassware"); card.setImageLayout(Card.ImageLayout.LEFT); View cardView = card.getView();

Quick Start Guide

Get started building your first Glassware application. This guide will walk you through the basics of both GDK and Mirror API development.

Prerequisites

  • Android Studio or Eclipse with ADT
  • Glass device or emulator
  • Android SDK (API 19)
  • GDK add-on

Setting Up Your Development Environment

1. Install the Android SDK

Download and install the Android SDK. Make sure you have API level 19 (Android 4.4) installed.

2. Add the GDK

The GDK is an add-on to the Android SDK. Install it through the SDK Manager:

SDK Manager → Tools → Manage Add-on Sites Add: https://developers.google.com/glass/gdk

3. Create Your First Project

Create a new Android project with the following settings:

  • Minimum SDK: API 19
  • Target SDK: Glass Development Kit Preview
  • Compile with: Glass Development Kit Preview

Hello World Example

public class MainActivity extends Activity { @Override protected void onCreate(Bundle savedInstanceState) { super.onCreate(savedInstanceState); Card card = new Card(this); card.setText("Hello, World!"); card.setFootnote("My First Glassware"); setContentView(card.getView()); } }

Voice Command Setup

Add a voice trigger to launch your Glassware:

<!-- AndroidManifest.xml --> <activity android:name=".MainActivity"> <intent-filter> <action android:name="com.google.android.glass.action.VOICE_TRIGGER" /> </intent-filter> <meta-data android:name="com.google.android.glass.VoiceTrigger" android:resource="@xml/voice_trigger" /> </activity>

Design Principles

Glass is fundamentally different than existing mobile platforms. Follow these principles when building Glassware to give users the best experience.

Design for Glass

Design for people

Build experiences that are genuinely human and focus on the user's needs. Don't get in the way.

Don't get in the way

Glass is designed to be there when you need it and out of the way when you don't. Your Glassware should follow this principle.

Keep it relevant

Deliver information at the right place and time. Use location and context to make your Glassware more useful.

Avoid the unexpected

Don't surprise users with unexpected content or functionality. Be predictable and reliable.

Build for the future

Glass is just the beginning. Design your experiences to work across different form factors and contexts.

Keep it simple

Glass users are often on the go. Keep interactions brief and interfaces simple.

The Four Cs

1. Current

Information should be relevant to the present moment and context.

2. Clear

Content should be easy to understand at a glance.

3. Concise

Keep text short and to the point. Users should be able to read content in 2-3 seconds.

4. Compelling

Make your content engaging and worth the user's attention.

Glass Development Kit (GDK)

The Glass Development Kit is an add-on to the Android SDK that lets you build Glassware that runs directly on Glass.

Key Features

  • Access to Glass-specific hardware
  • Live Cards for ongoing tasks
  • Immersions for full-screen experiences
  • Voice and gesture input
  • Direct sensor access

Live Cards

Live Cards display information in the timeline and can update frequently. They're perfect for ongoing tasks.

public class LiveCardService extends Service { private LiveCard mLiveCard; @Override public int onStartCommand(Intent intent, int flags, int startId) { if (mLiveCard == null) { mLiveCard = new LiveCard(this, "my_card"); RemoteViews remoteViews = new RemoteViews( getPackageName(), R.layout.live_card); mLiveCard.setViews(remoteViews); Intent menuIntent = new Intent(this, MenuActivity.class); mLiveCard.setAction(PendingIntent.getActivity( this, 0, menuIntent, 0)); mLiveCard.publish(PublishMode.REVEAL); } return START_STICKY; } }

Immersions

Immersions give you full control over the user experience, perfect for games or complex interactions.

public class ImmersionActivity extends Activity { private GestureDetector mGestureDetector; @Override protected void onCreate(Bundle savedInstanceState) { super.onCreate(savedInstanceState); setContentView(R.layout.immersion_layout); mGestureDetector = createGestureDetector(this); } private GestureDetector createGestureDetector(Context context) { GestureDetector gestureDetector = new GestureDetector(context); gestureDetector.setBaseListener(new GestureDetector.BaseListener() { @Override public boolean onGesture(Gesture gesture) { if (gesture == Gesture.TAP) { // Handle tap return true; } else if (gesture == Gesture.SWIPE_RIGHT) { // Handle swipe right return true; } return false; } }); return gestureDetector; } }

Mirror API

The Google Mirror API allows you to build web-based services that interact with Google Glass. It provides this functionality over a cloud-based API that doesn't require Android code.

Key Concepts

Timeline Items

Timeline items are the primary way to deliver information to users. Each item is a card that users can interact with.

{ "text": "Hello Glass!", "notification": { "level": "DEFAULT" }, "menuItems": [ { "action": "REPLY" }, { "action": "DELETE" } ] }

Subscriptions

Subscribe to notifications when users interact with your timeline items or when their location changes.

POST https://www.googleapis.com/mirror/v1/subscriptions { "collection": "timeline", "userToken": "user_1234", "callbackUrl": "https://yourserver.com/notify" }

Contacts

Contacts allow users to share timeline items with your Glassware.

{ "id": "myapp_contact", "displayName": "My App", "imageUrls": [ "https://yourserver.com/icon.png" ], "acceptCommands": [ { "type": "POST_AN_UPDATE" } ] }

Authentication

The Mirror API uses OAuth 2.0 for authentication. Users must authorize your application to access their Glass.

OAuth 2.0 Flow

  1. Direct users to Google's authorization endpoint
  2. User grants permission
  3. Receive authorization code
  4. Exchange code for access token
  5. Use access token for API calls

Voice Input

Voice input lets you create a truly hands-free interface. Glass gives you three ways to utilize voice input.

Voice Triggers

Main voice commands start Glassware from the Home card.

<!-- res/xml/voice_trigger.xml --> <trigger keyword="@string/glass_voice_trigger"> <input prompt="@string/glass_voice_prompt" /> </trigger>

Voice Menu Commands

Contextual voice commands appear in the "ok glass" menu.

@Override public boolean onCreateOptionsMenu(Menu menu) { getMenuInflater().inflate(R.menu.main, menu); return true; } @Override public boolean onOptionsItemSelected(MenuItem item) { switch (item.getItemId()) { case R.id.action_share: // Handle share action return true; default: return super.onOptionsItemSelected(item); } }

Free-form Speech Input

Prompt users to speak and convert their speech to text.

Intent intent = new Intent(RecognizerIntent.ACTION_RECOGNIZE_SPEECH); startActivityForResult(intent, SPEECH_REQUEST); @Override protected void onActivityResult(int requestCode, int resultCode, Intent data) { if (requestCode == SPEECH_REQUEST && resultCode == RESULT_OK) { List<String> results = data.getStringArrayListExtra( RecognizerIntent.EXTRA_RESULTS); String spokenText = results.get(0); // Process the spoken text } }

Best Practices

Follow these best practices to ensure your Glassware provides the best user experience.

Performance

  • Keep your APK under 50MB
  • Optimize images for 640×360 display
  • Use efficient data structures
  • Minimize battery usage
  • Cache data when possible

User Interface

  • Follow Glass style guidelines
  • Use high contrast colors
  • Keep text concise (40 characters or less)
  • Use the standard Glass fonts
  • Provide clear navigation

Privacy

  • Be transparent about data collection
  • Respect user privacy
  • Follow the Glass Platform Developer Policies
  • Don't record without permission
  • Protect user data

Testing

  • Test on actual Glass hardware
  • Test in different lighting conditions
  • Test while walking
  • Test with different head positions
  • Test battery consumption
